# Changelog

All notable changes to this project are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Added
- Adopted **beads (`bd`)** as the Dolt-backed task/memory layer beneath GitHub Issues ([#152](https://github.com/J-MaFf/gitconfig/issues/152)). The bead graph syncs via `refs/dolt/data` on `origin`; `.beads/config.yaml` and `.beads/metadata.json` are tracked while the JSONL exports and embedded Dolt DB are gitignored (dolt-only sync).

### Fixed
- Dead mid-test `! grep` assertions in the shared bats suites were silently swallowed: bats exempts a `!`-negated command from its errexit/ERR-trap machinery, so only a *terminal* negation fails the test — the mid-test `! grep` checks in `mac-initialize-local-config.bats` (and one in `functions.bats`) never asserted anything, and the rest passed only by accident of placement. Converted all 13 negated-grep assertions across `functions.bats`, `mac-` and `linux-initialize-local-config.bats` to `run ! grep` (position-independent, bats >= 1.5) and added a `bats_require_minimum_version 1.5.0` pragma to each ([#194](https://github.com/J-MaFf/gitconfig/pull/194)).
- `git skill publish` on Windows died on `publish-skill.ps1`'s `#Requires -Version 7`: the dispatcher (`_run_skill_script` in `gitconfig_helper.py`) launched every wrapper script with Windows PowerShell 5.1, which refuses to start PS7-only scripts even when pwsh 7 is installed. It now resolves `pwsh` first, falls back to `powershell` only for the WinPS-compatible sync/status wrappers, and prints an actionable install hint when a PS7-only script is dispatched without pwsh on PATH ([#189](https://github.com/J-MaFf/gitconfig/pull/189))
- The Windows-generated `~/.gitconfig.local` now sets `core.longpaths = true`: Dolt's git-remote-cache (behind beads' `bd dolt push`) nests refs past Windows MAX_PATH, and stock Git for Windows fails with "Filename too long". The setting had been hand-applied to the global config on 2026-07-02, but template regeneration (`git selfupdate`) rewrites `~/.gitconfig` and silently wiped it — it regressed unnoticed until 2026-07-08. It now lives in the never-clobbered local config (Windows-specific, so the shared template stays platform-neutral), with a Pester regression guard ([#187](https://github.com/J-MaFf/gitconfig/pull/187))
- Running the wrong platform's `install.sh` displaced the machine's correct `~/.gitconfig.local`: STEP 0 (`cleanup-gitconfig.sh --force`) moved it to a backup before the platform guard inside `initialize-local-config.sh` (STEP 3) could fire, leaving the install aborted mid-run. Both `install.sh` wrappers now carry the same `uname -s` guard (with the `GITCONFIG_ALLOW_CROSS_OS=1` escape hatch) before any destructive step, with a `tests/shared/install-platform-guard.bats` suite pinning it ([#185](https://github.com/J-MaFf/gitconfig/pull/185)).
- The mac `initialize-local-config.sh` credential block now emits the empty `helper =` reset line before `osxkeychain`, matching the Linux blocks from #180: `credential.helper` is multi-valued and accumulates across config levels, so a stale helper inherited from a migrated other-platform `~/.gitconfig.local` would otherwise answer first ([#184](https://github.com/J-MaFf/gitconfig/pull/184)).
- The generated `~/.gitconfig.local` on Linux carried no HTTPS credential helper — and the Ubuntu server's was a macOS-flavored file applying `credential.helper = osxkeychain`, which broke every HTTPS git operation (`bd dolt push`, private-repo pulls: "git: 'credential-osxkeychain' is not a git command"). The Linux initialize script now emits a platform-appropriate `[credential]` block (gh CLI helper scoped to github.com/gist.github.com with an empty reset line so stale inherited helpers can't answer first, libsecret fallback, commented hint otherwise), and both the mac and linux `initialize-local-config.sh` refuse to run on the wrong OS (`GITCONFIG_ALLOW_CROSS_OS=1` escape hatch for the sandboxed bats suites) ([#180](https://github.com/J-MaFf/gitconfig/pull/180)).
- The Windows `Initialize-LocalConfig.ps1` config-verification test failed when the suite ran on macOS: the script builds `USERPROFILE`-relative paths with backslashes, so on Unix hosts verification read a mixed-separator path and warned spuriously. The test now skips on non-Windows hosts (discovery-safe check that still runs under Windows PowerShell 5.1), making the suite fully green on macOS ([#178](https://github.com/J-MaFf/gitconfig/pull/178)).
- `tests/gitconfig_helper.Tests.ps1` hardcoded `python` (28 failures on macOS, where the interpreter is `python3`) and built git fixtures from the Windows-only `$env:TEMP` with unguarded `Push-Location`/`Pop-Location` — on macOS a failed fixture setup let the tests' git commands run against the developer's checkout (committing fixture data and renaming the checked-out branch to `main`). The interpreter now resolves `py` → `python3` → `python` (whole file skips cleanly when absent), fixtures use the cross-platform temp root with `-ErrorAction Stop`, and location pops are guarded ([#177](https://github.com/J-MaFf/gitconfig/pull/177)).
- The shared beads Dolt DB (schema v32) was unusable from bd 1.1.0 clones, and the upstream migration chain is broken for this lineage (migration 0047 references a `wisps` table that never existed). Re-initialized the graph fresh on the native schema — the old graph was verified empty at every version — and replaced `refs/dolt/data` on origin. Other machines must upgrade `bd` and re-run `bd bootstrap` ([#175](https://github.com/J-MaFf/gitconfig/pull/175)).
- Regenerating `~/.gitconfig.local` on a shared Mac wiped the hand-added `safe.directory = /opt/homebrew` entry, so git's dubious-ownership check broke `brew update`. The mac initialize script now emits the entry whenever the Homebrew repo exists and is owned by a different user ([#170](https://github.com/J-MaFf/gitconfig/pull/170)).
- Regenerating `~/.gitconfig.local` on the Mac reverted commit signing to the template's agent-based 1Password key (Touch ID prompt per commit; hangs unattended runs). The mac initialize script now detects an on-disk signing key (`~/.ssh/claude_desktop` or `~/.ssh/id_ed25519_signing`) and emits the file-based signing block — including the `git-sign-no-agent` wrapper when present — preferring it over `op-ssh-sign` ([#173](https://github.com/J-MaFf/gitconfig/pull/173)).
- Windows login scheduled task logged "No Python interpreter found" even when Python was installed: `Resolve-Python` picked the 0-byte WindowsApps app-execution-alias stub, which only resolves in an interactive session. It now skips those stubs and falls back to the real PyManager/launcher path ([#161](https://github.com/J-MaFf/gitconfig/pull/161)).
- Non-ASCII em-dashes (U+2014) in `Update-GitConfig.ps1` and `gitconfig_helper.py` failed the ASCII Encoding tests (Windows PowerShell 5.1 compatibility) ([#164](https://github.com/J-MaFf/gitconfig/pull/164)).
- The Pester suite mutated the real machine (overwrote `~/.gitconfig`/`~/.gitconfig.local`, toggled the login scheduled task): `run-tests.ps1` set tag filters under the wrong Pester config section so `-ExcludeTag Integration` was ignored, and two test files generated config against the real `$env:USERPROFILE`. Tests now sandbox HOME and integration tests are excluded by default ([#167](https://github.com/J-MaFf/gitconfig/pull/167)).

### Changed
- `git skill sync` / `git skill status` now render in Python with rich instead of dumping the wrapper scripts' raw text ([#191](https://github.com/J-MaFf/gitconfig/pull/191)). The old output replayed the last *scheduled* sync's log tail — including its stale `[drift]` flag and the CLAUDE.md `1 file(s) copied.` noise — next to freshly computed state, which read as a contradiction, and printed identical before/after blocks around the pull. The new view keeps only what stays true from the log (last run's stamp + pull result), computes drift once and live via `skill-audit` (still the single source of truth), colors the counts (green aligned / yellow drift / red problems), warns when the repo isn't on `main`, and turns the gone-upstream pull failure into an actionable "switch back to main" hint. The claude-skills wrappers remain for direct use and as the automatic fallback if the rich view fails.
- `install.ps1` no longer overwrites an existing `~/.gitconfig.local` (create-if-missing). The machine-specific config is preserved on re-install; regenerate deliberately with `Initialize-LocalConfig.ps1 -Force` ([#166](https://github.com/J-MaFf/gitconfig/pull/166)).
- `tests/run-tests.ps1` excludes integration tests by default; opt in with `-IncludeIntegration` (run only on a real machine or VM) ([#167](https://github.com/J-MaFf/gitconfig/pull/167)).
